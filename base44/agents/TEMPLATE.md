# Name

The employee's title/role (e.g. "CTO", "Backend Engineer", "QA Engineer").

# Mission

One or two sentences: the single overarching purpose this employee exists to serve within Decifra.

# Role

A short paragraph describing what this employee is (and, if useful, what they are not — e.g. "never writes code"), and where they sit in the organization.

# Responsibilities

A bullet list of the concrete duties this employee owns. Each bullet should be a short, bolded lead-in followed by one sentence of explanation.

# Inputs

What this employee receives before they can act (e.g. an approved GitHub Issue, an implementation plan, a business goal, output from another employee).

# Outputs

What this employee produces (e.g. a task breakdown, a pull request, a review report, a test plan).

# Workflow

A numbered, step-by-step sequence describing exactly how this employee handles a request from start to finish.

# Decision Principles

An ordered list of priorities this employee must weigh when making decisions, plus any hard rules about what must never be sacrificed for what.

# Rules

Hard constraints this employee must always follow, stated as absolute "never/always" statements.

# Definition of Done

The concrete, checkable conditions that must all be true before this employee's work is considered complete.

# Success Metrics (KPIs)

The measurable indicators used to evaluate whether this employee is performing well.

# Collaboration

Which other AI employees this role works with, and what each handoff looks like.

# Escalation

When this employee must stop and ask the CTO or the human for approval before proceeding.
